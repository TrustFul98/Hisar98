Folgende Technolgien / Frameworks setze ich in meinem Projekt ein:

1. JavaScript/ Canvas
2. HTML
3. CSS

zu 1: 	Ich habe mich für 1. entschieden, weil mir der Umgang mit JavaScript sehr leicht fällt. JavaScript ist relativ unkompliziert.
	Auch wenn mit der Menge an Code die Übersicht erschwert wird finde ich dennoch, dass JavaScript echt super funktioniert. JS funktioniert
	im Web problemlos und somit auf allen Betriebssystemen. Was bedeutet das? Das heißt mit JacaScript werden alle Nutzer angesprochen!
	Klar kann man dieses Beispiel auch mit Java zum laufen bringen beispielweise via WindowBuilder, was auch absolut super ist, dennoch finde ich
	und das können Sie bestimmt bestätigen, dass Anwendungen im Internet viel leichter mit JavaScript zu handlen sind.
	Canvas: Das Problem ist, ich habe sonst keine Möglichkeit gefunden ein DIV-Element von HTML in eine Bilddatei umzuwandeln. Nach
	meiner Recherche bin ich auf html2canvas gestoßen. Die Bibliothek heißt "html2canvas.js". Hier habe ich das DIV-Tag auf das canvas
	gemalt und in ein anderes DIV-Element integriert. Von diesem DIV-Tag habe ich die DateiURL genutzt, um href vom a-Tag zu definieren.
	Innerhalb des a-Tags habe ich die download option integriert. Dank Canvas habe ich nun mein Ziel erreicht.
	
zu 2: 	Da ich mich in letzter Zeit intensiver mit Webentwicklung beschäftigt hatte, fiel meine Auswahl ohne zu überlegen auf HTML/ CSS.
	Oberflächenprogrammierung ist mit HTML/ CSS ein einzigartiges feeling und nicht zu ersetzen. Mir macht das einfach tierischen Spaß
	direkt zu sehen, was Schritt für Schritt passiert.

zu 3: 	CSS ist zum designen der HTML-Tags einfach unbeschreiblich. Die Möglichkeiten mit CSS das Layout zu frisieren ist ebenfalls echt top.
	Animationen leicht gemacht, 3D-Effekte leicht gemacht, Bearbeitungsmöglichkeiten auf zig verschiedene Wege Möglich!

Eigentlich kann man alles in einem Paket einschließen, weil alles perfekt aufeinander abgestimmt ist. Für mich absolut perfekt.

Wie schon erwähnt habe ich html2canvas.js genutzt, was auch unbedingt notwendig war. Als auch fonticons zum Ansprechen des Nutzers.
